Overview

Light Days is a native app developed for both iOS and Android. The app is for all women who want to keep an eye on their periods and get to know or better understand their body. Especially for women who suffer from their periods and want to receive suggestions to alleviate symptoms and better plan their daily life.

Problem

There are many apps that show the menstrual cycle and help with family planning. With most apps, symptoms can be entered, but there is no app that offers suggestions for solutions and is also well-organised.

Solution

A simple app that allows the user to see the current status of her cycle. The app allows the user to get information about symptoms directly and save them.

Objective

The aim of the app is to allow the user to navigate through the app quickly and easily. In addition, the app should have a feminine style. The user should get a good overview of her menstrual cycle and be able to enter her symptoms. The user receives suggestions for relieve prevention and treatment of symtoms, which can be saved if desired. If the user has a desire to have children, she receives information about her fertile days.

When will my audience use the app?

  • Start and end of periods
  • For complaints | symptoms
  • For planning rest times, holidays and sports activities
  • To support family planning | desire to have children

Where will they use the app?

Anytime, anywhere the user has her mobile phone available.

User Flow

The following tasks can be completed by the user when using the app:

  • Create a profile
  • Edit period data
  • Add ovulation
  • Add ovulation
  • Identify fertile days
  • Identify symptoms
  • Get information about symptoms
  • Suggestions for prevention | treatment
  • Save symptoms | information

Learning from others

In the next step, I analysed three competitors. I focused on their strengths, which I can use for my app. Then I applied my decisions to the mid-fidelity wireframes in Figma.

  • Great infographic that awakens interest
  • Clear design, well structured
  • Cycle phases clearly displayed
  • Nice setting list
  • A clear, structured home screen
  • Calendar shows period, fertile days, and ovulation clearly
  • Great calendar view and structure
  • Complaints are structured in moods and symptoms
  • Selection options are immediately clear
  • Filter page appears structured, clean and harmonious
  • Cycle phases differentiated by colour

High-Fidelity Wireframes and Prototype

Once the hierarchy and patterns were set, I went on to create the high-fidelity wireframes. After that I built a prototype to test the functions and design on my test users.

User Feedback

I tested the high-fidelity wireframes on my test users and I used the negative criticism for my next iteration and to improve and then to finalise my design.

  • The element borders on all pages seem too close

Luna - Android

Final UI

After the user feedback, I made some changes to the UI and also to the UX part and designed the final UI. This includes the complete visual design like typography, iconography and colour.

Visual Design

The visual design is in muted colours and very feminine. The design is restrained and calming. The primary and secondary buttons are clear, making it easy for the user to navigate through the app.

Retrospective

What went well

The user feedback went well because I received feedback very quickly and could use it to improve the further design.

What can be improved

My Figma skills like auto layout, colour and effect styles, prototyping (there was always something that didn't work) and my knowledge of iOS and Android guidelines. It wasn't easy to decide what was optional and what was required. Solution: To improve my skills I watch tutorials every second day.

What went wrong

My time management. I had sleepless nights, because of my user feedback. I made too many changes too early and that created even more work Solution: Reconsider user feedback and don't adapt every request. Use error rating!
